Install

Pick the path your team will actually use.

The Python package is primary. The other routes lower discovery friction for teams that start from shell snippets or JavaScript tooling. 

python -m pip install git+https://github.com/lekhanpro/gitmortem.git
export GROQ_API_KEY=your_key
gitmortem HEAD~1

After the first tagged PyPI release, the install line becomes pip install gitmortem.

curl -fsSL https://raw.githubusercontent.com/lekhanpro/gitmortem/main/install.sh | bash
gitmortem HEAD~1 --no-llm

Useful when the fastest path is a shell install and an offline first run.

irm https://raw.githubusercontent.com/lekhanpro/gitmortem/main/install.ps1 | iex
python -m gitmortem HEAD~1

A Windows-friendly path that keeps the same Python runtime underneath.

npm install -g github:lekhanpro/gitmortem
gitmortem HEAD~1

The npm package is a wrapper for discovery and convenience. The Python CLI remains the core runtime.

FAQ

Short answers before you try it on a real incident.

Does this send my whole repository to a model?

No. The tool is built around commit facts, diff summaries, nearby history, and dependency hints rather than blindly shipping an entire repository snapshot by default.

Can I use it without any API key?

Yes. --no-llm produces a deterministic report based on git metadata, parsed diffs, blast-radius heuristics, and surrounding commit history.

Am I locked into Groq?

No. Groq is the default because it makes the CLI feel fast, but provider choice stays configurable at runtime.

Who is this actually for?

Maintainers investigating a bad release, platform engineers estimating blast radius, and teams who want a cleaner first pass before writing the formal postmortem.
